Written Questions


1. My dashboard processed relevant variables from the given netflix dataset in order to plot data pertaining to every question in a presentable manner. This way, I was able to clearly answer the provided questions.
Here, I chose to use 2 barplots, for graphs 1 and 3, wherein I grouped the assigned the independent variables to the y-axis and dependent variables to the x-axis, ensuring to label data counts for every bar for easy readibility.
Next, I chose to plot a line graph for graph 2, with the grouped independent variable on the x-axis and dependent variable on the y-axis. For this graph, instead of assigning data labels, I chose to use an interactive mouseover feature which allows the user to hover over the plot in order to access the data values.
Clearly labeling the chart titles and axis helped make the graphs easy to understand.
2. D3 was helpful in improving my visualization for the following reasons:
i) This tool helped me code interactive charts that are easy to understand for viewers by customizing everything- from axis scales to colour combinations.
ii) It was a tool that linked data and graphics in the most user-friendly way. Data visualization is the optimal tool to help communicate data insights, and D3 made every step of this easy to follow.
iii) Given the tool's popularity, I was able to guide my process through a host of tutorials that helped me truly internalize the power of it and harness its every benefit.
3. D3 might not be the best tool for creating a visualization for the following reasons:
i) This tool doesn't generate predetermined visualizations. Resultantly, it can be time consuming to use this while online tools can compile a basic chart much faster.
ii) It has a steep learning curve. It took me a significant amount of time to get comfortable with the syntax and tools offered by D3, which make it overwhelming to start with.
iii) I tried flipping the axis for graph 3's barplot, with the independent variable on the x-axis instead of on the y, however this caused several complications that weren't previously explored by the D3 community online.
4. My dashboard can be easily accessed by people with say, sound or verbal impairments as it does not harness audio features. However, it doesn't meet the needs of people with visual impairments. I order to make this more accessible to them, I could include a text based write-up explaining the insights derived from the data and its corresponding visualization in a manner that it can be dictated to them and hence accessed by them. To address people with colour blindness, I recognize that the colour palette of my design is largely green. However, seeing this colour any differently does not alter the essence of what I am trying to convey. That being said, I could have made use of a more conscious colour.
